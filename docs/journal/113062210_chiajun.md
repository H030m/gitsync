**做了什麼**（四個 trellis 工作，皆已 archive；分支 `feature/discord-range-cursor`，已 push）

   - `listDaySummaries` / `getDaySummary` — 讀**每日 digest**（省 context；摘要類問題先看這個，不必讀原始訊息）。

2. **省 context 設計**：因為逐日 digest 已存在，AI 預設讀 day summary，只有要精確引用才讀原始訊息（prompt「最省成本優先」）。
3. **digest 卡片 收合 / 鎖定 / AI 改寫**（與 opal 協作）：`_DigestCard` 改成可收合（動畫）、加**鎖定鈕**（鎖住後擋掉所有自動覆寫——每日排程 + AI 改寫都跳過，狀態存 `discordDigests/{date}.locked`）、卡片內「叫 AI 調整這份總結」欄。新 callable `editDiscordDigest` / `setDigestLock` / `botEditDigest`（bot 橋接）+ Discord slash command **`/gitsync-digest`**。
4. **範圍雙 cursor 回補 + prune**（`06-03-discord-range-cursor`）：把「單一起始日期」改成 **`[startDate, endDate]` 範圍**，存 `repos/{repoId}.discordStartDate/EndDate`。衍生兩個 snowflake cursor：low = `snowflakeForTaipeiDate(start)`、high = `snowflakeForTaipeiDayEnd(end)`（exclusive）。bot 抓 `after: lastMessageId ?? low` 且**忽略 id ≥ high**。新 callable `setDiscordRange` 寫範圍 + reset 各頻道 watermark + **prune**（刪範圍外的 `discordMessages` 與 `discordDigests`）。**逐日 digest** `discordRangeDigestFlow`：對範圍內每天各產一份 digest（空白/鎖定/未變動的天用 `count()` 跳過，不呼叫 OpenAI；上限 92 天）。
5. **分組 snippet + Updated 提示**（`06-03-discord-snippets-updated`）：`searchDiscordMessages` 改回傳**對話片段**——每個命中訊息前後各帶 2 則同頻道上下文（`isMatch` 標記），相鄰命中視窗合併，依命中數→時間排序，回傳 `{answer, snippets}`；UI 把每段渲染成叢集（命中強調、脈絡淡化）並用 divider 分隔。Refresh 改成**等真正完成**才停 spinner（串 `fetchRequests/{id}.status` 到 terminal），完成跳 `Updated ✓` + 顯示 `Updated HH:mm`。
6. **修補**：小視窗聊天框 overflow（空狀態改可滑動）、digest 展開內容過長可滑動、bot 加 per-channel cursor 診斷 log。

**為何這樣做**

- **為何省 context 走逐日 digest 而非每次都餵原始訊息**：訊息量大時直接餵 transcript 會爆 token；先讀「每天一句摘要」再決定深讀哪天，context 從 O(messages) 降到 O(days)。
- **為何雙 cursor**：舊單 watermark 在「已讀到 7 日、想把起點改回 5 日」時會被 `watermark(7) > snowflake(5)` 卡住抓不到前面；low/high 兩指針 + reset watermark + prune 才能正確縮放範圍。
- **為何 prune 是破壞性的**：使用者明確要「超出範圍的訊息與日報刪掉」；範圍放寬時 bot 會重抓（messageId 去重保證不重複）。
- **為何 related message 要分組帶上下文**：單看命中那一句看不出脈絡（誰回了什麼），帶前後文才讀得出「i push a commit → ok」「the commit is broken → i'll fix」這種來回。
- **related message 目前仍是關鍵字子字串比對（非語意）**：Discord 訊息還沒做 embedding（`onDiscordMessageCreated` 仍 stub），所以中文比對偏弱、會漏同義詞。
